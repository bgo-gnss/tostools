# Destination: `tos audit attribute-dates` — detect misdated TOS attribute periods

## Problem

TOS attribute periods can be entered retroactively, and the database auto-stamps
the period `date_from` with the *data-entry* date rather than the date the value
was first applicable to the device. Concrete case: ARHO Ashtech (id 4520-ish)
has `Raðnúmer = 13831` with `Frá 2014-10-17` even though the receiver lived at
the station from 2002-01-01. The new synthesis chain correctly renders TOS, so
the phantom date propagates into PrintTOS / sitelog / GAMIT outputs. The fix has
to happen in TOS data, but we need a way to *find* the pattern across 173
stations rather than stumbling on it one station at a time.

## In scope

- New CLI verb `tos audit attribute-dates <STA>` — per-station, default GPS quartet
  subtypes (`gnss_receiver`, `antenna`, `radome`, `monument`), `--subtypes` to widen.
- **Rule 3:** flag any attribute period whose `date_from` is later than the device's
  earliest known period (min over earliest attribute period + earliest join period).
- **Inherent vs mutable** attribute taxonomy lives in `data/attribute_codes.yaml`,
  one entry per code with classification + a `why` comment. Default rule runs on
  inherent codes only; `--include-mutable` widens.
- Outputs: text report to stdout, `--triage <path>` emits draft ACTION file
  (commented-out by default), exit code 0/1 for cron friendliness.
- Suppression file `data/audit_suppressions/attribute_dates.txt` — `ACTION`-style
  line format, granular by `(id_entity, code, date_from)`, committed in repo.
- Catalog YAML built by walking live TOS for a GPS-quartet device sample, harvesting
  every distinct code seen, emitting draft entries for operator review.

## Out of scope (v1)

- Auto-applying corrections. Detection emits triage; operator runs `tos audit apply`.
- Synthesis-layer masking. Rule 3 doesn't run during PrintTOS / sitelog rendering.
- Auto-suggesting `date_from` for mutable attributes (just flag).
- TOS schema additions for a "reviewed-OK" marker.
- Fleet sweep (`--all`) — follows once per-station verb is validated.
- Cross-domain expansion (meteorological / seismic / hydro) — opt-in via `--subtypes`.
- Web / dashboard view.

## Definition of done (4 layers)

1. `data/attribute_codes.yaml` exists, every GPS-quartet code we see is classified
   `mutable | inherent` with a comment explaining why.
2. `tos audit attribute-dates ARHO` flags the misdated SN period (ARHO Ashtech,
   `serial_number`, 2014-10-17). Does NOT flag firmware bumps. Exit code 1 when
   anything's flagged, 0 when clean.
3. `data/audit_suppressions/attribute_dates.txt` is read at audit time; adding a
   `SUPPRESS <id> <code> <date_from>` line removes that entry from the next run.
4. End-to-end: `--triage` emits draft ACTION file → operator edits + applies via
   `tos audit apply --apply` → TOS reflects corrected `date_from` → re-running the
   verb shows ARHO clean.

## Hard constraints

- Don't regress 380/2 test suite.
- Catalog walk needs live TOS read access (no cassette infra yet for the harvest).
- New verb plugs into existing `_audit_main` dispatcher; no parser refactor.
- Suppression file: `ACTION`-style line format, not JSON / YAML.
- Catalog file: YAML for per-code comments.
